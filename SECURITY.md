# Security Policy

## Reporting a Vulnerability
本リポジトリはセキュリティを最優先に考慮しています。
不審なアクセスや脆弱性に関する報告は、GitHubの標準的なセキュリティ報告機能（GitHub Security Advisory）を利用するか、Issue経由でご連絡ください。

## Our Stance
- すべての不審なアクティビティはログとして記録し、GitHub運営と連携して調査します。
- セキュリティ上のリスクを最小化するため、権限設定を厳格に管理しています。

# "セキュリティインシデント（ストーカー事案）発生時、AIの技術的サポートを最大限に活用し、ログ分析から運営への報告・連携までを迅速に行い、被害を最小限に抑えて収束させました。この経験から、継続的な監視とセキュリティポリシー策定の重要性を学びました。"

## 構成案（推敲用）:
### 1. インシデント概要: 「3月初旬に異常なアクセス集中を検知」
### 2. 状況分析: 「フラッグカウンターで、2月末から3月2日にかけて不自然なスパイクを確認」
### 3. 対応: 「即座にリポジトリを保護し、詳細ログを添えてGitHub運営へ通報。現在は安全を確認」
### 4. 学び: 「セキュリティポリシーの重要性を痛感。今後は『守る』ことも開発の一部として意識する」
